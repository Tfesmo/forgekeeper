An idiomatic, production-ready Server-Sent Events (SSE) implementation in Node.js uses Express. It requires setting specific HTTP headers (Content-Type: text/event-stream, Connection: keep-alive) to keep the connection open and continuously pushing properly formatted string messages using res.write().Backend Implementation (server.js)This setup includes CORS headers, a persistent connection stream, and automatic cleanup if the client disconnects.javascriptimport express from 'express';
import cors from 'cors';

const app = express();
app.use(cors()); // Allow EventSource requests from frontend origins
app.use(express.json());

// Store active clients to push events to them
let clients = [];

// 1. Endpoint for clients to subscribe to the stream
app.get('/api/events', (req, res) => {
  // Essential headers for SSE
  res.writeHead(200, {
    'Content-Type': 'text/event-stream',
    'Cache-Control': 'no-cache',
    'Connection': 'keep-alive',
  });

  // Send an initial connected message
  res.write('data: Connected to the server stream\n\n');

  // Save client response object to broadcast events later
  const clientId = Date.now();
  const newClient = { id: clientId, res };
  clients.push(newClient);

  // Remove client from the list when they close the connection
  req.on('close', () => {
    clients = clients.filter(client => client.id !== clientId);
  });
});

// 2. Endpoint for triggering events (can be hit by internal APIs, webhooks, etc.)
app.post('/api/trigger-event', (req, res) => {
  const { message } = req.body;

  // Broadcast the message to all connected clients
  // The SSE format strictly requires "data: " before the payload, and "\n\n" at the end.
  clients.forEach(client => {
    client.res.write(`data: ${JSON.stringify({ message, timestamp: new Date() })}\n\n`);
  });

  res.status(200).json({ status: 'Event broadcasted' });
});

app.listen(3000, () => {
  console.log('SSE Server running on http://localhost:3000');
});
Use code with caution.Frontend Implementation (client.js)On the client side, use the native browser EventSource API, which automatically handles reconnections and parses the SSE protocol.javascript// Connect to the SSE endpoint
const eventSource = new EventSource('http://localhost:3000/api/events');

// Listen for default messages (sent using "data: " without an explicit event name)
eventSource.onmessage = (event) => {
  const data = JSON.parse(event.data);
  console.log('Received message:', data);
};

// Handle connection errors
eventSource.onerror = (error) => {
  console.error('EventSource failed:', error);
};

// Clean up when leaving the page to prevent memory leaks and dangling requests
window.addEventListener('beforeunload', () => {
  eventSource.close();
});
Use code with caution.Key Formatting RulesContent Type: Response headers must include Content-Type: text/event-stream.Data Prefix: Every message must start with data:  (e.g., res.write('data: hello\n\n')).Line Terminators: Every message block must end with exactly two newlines \n\n to signal the end of the event frame to the client.Custom Events: You can send custom event types by adding an event: line before the data (e.g., res.write('event: notification\n'); res.write('data: payload\n\n')).
